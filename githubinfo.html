All about GitHub

  • Q: What is a version control system?
  • A: A system for organizing, tracking, and controlling different versions of files.
  • Q: What is Git?
  • A: The most widely-used version control system
  • Q: What is GitHub?
  • A: A platform that hosts Git repositories on the web
  • Q: What is the difference between Git and GitHub?
  • A: Git is the version control system itself, whereas GitHub is a web platform for facilitating its use
  • Q: Who started GitHub and how was it started?
  • A: Github was a start-up created by Tom Preston-Werner, Chris Wanstrath, and P.J. Hyett
  • Q: What company owns it now?
  • A: Microsoft
  • Q: How much does a GitHub account cost?
  • A: $0
  • Q: What is the Octocat?
  • A: The mascot of GitHub, which comes in various forms

Term definitions

  • Repository: A collection of related files stored on GitHub
  • Commit: A group of changes made to a repository's files
  • Fork: An alternate version of a repository created by copying the files
  • Push: The act of saving local changes to the remote branch
  • Pull Request: A request to merge changes from one branch to another
  • Workflow: A configurable automated process that will run one or more jobs
  • Issues: User-submitted error reports et cetra
  • Raw Button: Shows the raw content of a file
  • Blame Button: Shows the line-by-line revision history of a file
